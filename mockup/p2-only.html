同工端設計原則

裝置規格

📱 目標裝置：手機（≥390px）/ 平板（≥768px）
👆 按鈕觸控目標 ≥ 48px，防止地下室站立時誤觸
🔤 字體 ≥ 16px，車牌用 mono 加粗，一眼辨識
🔋 地下室 Wi-Fi 環境，需評估離線降級策略

同工可做什麼

  • ✅ 到場點名（大按鈕，一鍵點擊）
  • ✅ 遲到補點名（10:30 後仍可補，免違規）
  • ✅ 登記現場散客車牌
  • ✅ 查看剩餘空位數（供現場判斷）
  • ✅ 結束本週點名（觸發結算）

🔒 隱私保護

同工只看到「姓名、車牌、⭐ 優先車位」。不顯示 P2 具體原因（行動不便 / 孕婦 / 幼兒 / 年長者）與違規分數。

⏰ 關鍵時間點

  • 10:20 — 系統自動 LINE 通知 P2 未到場者
  • 10:30 — P3 逾時自動釋出（此版無 P3，同工知即可）
  • 10:45 — P2 逾時釋出（未回「正在路上」者）
  • 10:55 — P2 grace 最終釋出（已回「路上」者）

🛡️ 防呆機制

12:30 系統提醒同工結束點名；14:00 仍未結束則通知 Admin 手動補結算，避免資料卡死。
