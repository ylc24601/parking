# Staff 現場頁 — 會議展示版（Static Demo）

這是 **教會同工會議討論用** 的靜態展示頁，用來讓現場停車同工先「看到、摸到」第一版
Staff 現場點名頁的樣子並給回饋。**不是真實系統。**

- 純靜態單頁（HTML / CSS / vanilla JS），**沒有後端、沒有 Supabase、沒有真實 API 呼叫、沒有登入**。
- 頁面上所有資料都是**假資料**（姓名已遮罩、車牌為虛構），**不含任何真實會友資料**。
- 「點名 / 補點名」只是前端模擬，按下去只改畫面狀態，不會寫進任何資料庫。
- 行動優先，針對 **390px 寬手機畫面** 最佳化。
- 頁面頂端固定顯示橫幅：**「展示版，不是真實系統」**。

## 它示範了什麼

- 容量摘要（已到 N / 總數）
- 狀態篩選 chip（全部 / 未到 / 已到 / 已釋出）
- 車牌**後四碼**即時搜尋
- `approved` 列 →「點名」按鈕
- `released_late` 列 →「補點名」按鈕
- `attended` 列 → 顯示到場時間
- `attended_after_release` 列 → 顯示時間 +「（補）」
- ⭐ 優先車位顯示（只給布林星號，**不顯示原因**）
- 「登記現場車輛」「結束當週點名」為 **disabled / Coming soon**（第一版不做）

## 它刻意不顯示什麼（隱私邊界，與正式系統一致）

P2 優先原因、違規分數、連續未到次數、電話、LINE ID、釋出截止時間（`release_deadline_at`）、
是否「正在路上」（`p2_on_the_way`）、牧養關懷資料 —— **同工一律看不到。**

## 在本機打開

```bash
open docs/staff-demo/index.html
```

（直接用瀏覽器開啟即可，不需要任何伺服器或安裝。）

## 部署到 Surge（給會議現場用手機掃看）

```bash
cd docs/staff-demo
surge
```

第一次使用需先 `npm install -g surge` 並登入。部署後會得到一個公開網址，可在會議用手機開啟。

## ⚠ 注意

- **請勿** 在這個展示頁放入任何真實會友資料（真實姓名、車牌、電話等）。
- 這只是外觀與流程討論用；真實點名請走正式系統（需 PIN 登入與後端授權）。
