> 用途：轉交／複製給**教會 LINE 官方帳號（OA）管理者**，說明 Phase 5A production-OA dry-run 需要的協助。

> 背景與整體規劃見 [go-live-readiness.md](go-live-readiness.md)。**本階段完全不對外發送任何訊息。**

弟兄姊妹平安 🙏

我正在為主日停車系統建置一套「移車通知」功能：當地下室某個車位需要車主移車時，系統可透過**教會 LINE 官方帳號（OA）**發訊息通知該車主本人，**不需要、也不會顯示任何人的電話或個人 LINE**，同工只需要在現場點一下即可。

目前功能分兩階段上線。這次要做的是**第一階段的「測試」**，想先確認「會友傳訊息綁定車輛」這一段能在教會正式 OA 上正常運作。以下是想請您協助的事項。

## 一、這次測試「會做」與「不會做」什麼

**會做：**
- 讓少數幾位同工（測試者）用自己的 LINE 傳一則指定訊息給教會 OA，系統把這筆「綁定申請」記錄下來。

**不會做（請放心）：**
- ❌ 不會發任何訊息給會友（本階段完全不送出通知）。
- ❌ 不會自動回覆會友的訊息。
- ❌ 不會取得或顯示任何人的電話、個人 LINE 或聯絡方式。
- ❌ 不會更動教會 OA 現有的圖文選單、貼文或好友資料。

## 二、需要您協助提供 / 設定的三件事

**1. 一組「Channel secret」（頻道密鑰）**
- 用途：讓系統確認收到的訊息**真的來自 LINE**（安全驗證）。
- 這**不是**能發訊息的權杖，僅供驗證收訊，風險低。
- 位置：LINE Developers 後台 → 設定 → Messaging API。

**2. 設定一個「Webhook 網址」並開啟**
- 我之後會提供一段網址，請填入後台並按「Verify（驗證）」確認連線正常。

**3.（建議，測試期間）暫時關閉 OA 的「自動回覆」與「歡迎訊息」**
- 原因：這是 LINE 內建、與系統無關的自動訊息；測試期間關閉可避免測試者收到混淆的自動回覆。
- 測試結束後可隨時恢復。若教會目前很依賴歡迎訊息，我也可以改在人少的時段測試、或討論折衷做法。

## 三、隱私與安全說明

- 系統只會記錄「這個 LINE 帳號想綁定停車」這件事，**綁定成功與否仍需經過人工確認**（下一階段才做）。
- 同工端**看不到**任何人的 LINE 帳號或個資；只有教會官方帳號會在真正需要移車時通知車主本人。
- 測試期間**完全不對外發送任何訊息**，可隨時喊停、立即生效。

## 四、測試成功後的下一步

確認「綁定申請」能正確記錄後，我才會規劃第二階段：**人工審核通過→正式建立綁定**，屆時搭配正式權杖，移車通知才會真正送出。每一步都會先跟教會確認再進行。
