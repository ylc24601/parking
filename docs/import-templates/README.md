# 會友名單匯入範本（中文欄位）

> 給幹事整理名單用。兩份範本，依需求擇一或並用。
> ✅ **已上線（Wave 0）**：匯入器支援**中文表頭自動對照**與**兩種格式自動辨識**——
> 有「優先序」欄＝全體名單、有「申請原因」欄＝P2 申請表；兩份中文範本可**直接上傳匯入**
> （`/admin/import`）。範例資料皆為**虛構**，非真實個資。

---

## 1. 會友簡易名單範本.csv —— 全體會友（多數 P3）

最精簡、最好「看清楚全體」的格式。

| 欄位 | 說明 | 必填 |
|------|------|------|
| 姓名 | 會友姓名 | ✅ |
| 手機 | 手機號碼（同一人多車請填同號碼） | ✅ |
| 車牌 | 車牌（自動轉大寫、去空白/橫線比對） | ✅ |
| 優先序 | `P3`＝一般會友；`P2`＝需資格（要填 P2事由）；`P1`＝全職同工（**匯入只建一般會友＋車牌，不建全職同工/本週 P1 身分，輪值另管**） | ✅ |
| P2事由 | 僅 P2 需填：行動不便／短期不便／幼兒同行／孕婦／長者同行 | 選填 |
| 備註 | 自由文字 | 選填 |

**一人多車**：同手機、同姓名，分成多列（每台車一列）。

> ⚠️ **手機開頭 0 會被 Excel 吃掉**：用 Excel／Numbers 打開或編輯本 CSV 時，`0912345678` 會被當數字自動變成 `912345678`（掉開頭 0）。因手機是會友身分主鍵，掉 0 會對不上。
> **匯入器已容錯**：9 碼「9XXXXXXXX」會自動補回開頭 `0`。**但**若 Excel 已把號碼存成**科學記號**（如 `9.12346E+8`），數值已被四捨五入、無法還原——匯入器會**明確標錯並要求**把「手機」整欄設為**文字格式**後重新匯出再匯入（Excel：選欄→儲存格格式→文字，或用純文字編輯器／Google 試算表）。
> （`+886`／`886` 前綴目前不支援，列 backlog；見 feature-triage #22。）

## 2. P2資格申請範本.csv —— P2 資格申請（沿用現有紙本申請表）

欄位較多，對應紙本「停車申請表」。**申請原因用代碼**：

| 代碼 | 意義 | 資格效期 | 必附 |
|------|------|----------|------|
| 1 | 行動不便（長期／身障） | 永久 | 行動不便者姓名 |
| 2 | 行動不便（短期） | 自申請日 +6 個月 | 行動不便者姓名 |
| 3 | 幼兒同行（未滿 5 歲）**或**孕婦 | 幼兒：生日 +5 年；孕婦：另計 | 幼兒→孩童姓名+生日；孕婦→備註寫「懷孕」 |
| 4 | 長者同行（75 歲以上） | 永久 | 長者姓名 + 生日 |

- 日期格式：申請日期 `YYYY-MM-DD`；生日 `YYYY/MM/DD`。
- 幼兒最多 3 位（孩童姓名/生日 1–3）。

## 已上線功能（Wave 0，見 docs/feature-triage.md #20/#21/#22）

- ✅ **中文表頭對照層**：中文欄名自動對照 canonical 欄位；「P2事由」中文→canonical p2_reason。
- ✅ **兩格式自動辨識**：有「優先序」＝全體名單、有「申請原因」＝P2 申請表；兩者皆有→擋下請只用一種。
- ✅ **簡易全體匯入**：P3/P1 免填事由；P2 需填有效事由（無證據日期者建立後標「待覆核」）。
- ✅ **手機容錯**：9 碼補 `0`、科學記號明確拒絕。（`+886`／`886` 仍為 backlog。）

> 匯入器**永不撤銷既有 P2 資格**：全體名單把某位既有 P2 標為 P1/P3 時，只會保留原資格並在報表提示（資格變更請走資格審查）。
